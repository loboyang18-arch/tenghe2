# 数据目录说明（P3 数据与仓库解耦）

本目录用于存放**原始数据文件**。仓库内**不随代码提交大体积数据**，仅保留本说明与 schema，便于在 DSW/本机/OSS 上统一路径与列约定。

## 推荐用法

- **本机 / 云机**：将 Excel/CSV 放到 `data/` 下，运行脚本时指定路径，例如：
  ```bash
  python scripts/run_rt_baseline.py --file data/山东-全年-带时间点.xlsx
  python scripts/run_exog_suite.py --excel data/山东-全年-带时间点.xlsx
  ```
- **阿里云 DSW**：可将 OSS 桶挂载到 `data/`，或把数据下载到 `data/`，同样通过 `--file` / `--excel` 指定 `data/xxx.xlsx`。
- **根目录兼容**：若数据仍在项目根目录，可不改脚本，继续使用默认 `--file "山东-全年-带时间点.xlsx"`（脚本默认会从当前工作目录读）。

## 数据 schema（列约定）

脚本通过列名或 CLI 参数识别字段，下表为**建议列名**（与自动检测逻辑一致）：

| 用途           | 列名（任选其一或通过参数指定） | 说明 |
|----------------|------------------------------|------|
| 时间           | `datetime` / `时间` / `日期时间` / `时间点` / `timestamp` | 15 分钟粒度时间戳 |
| 实时出清电价   | `实时出清电价` / `实时出清价格` / `实时电价` / `RT出清电价` 等 | RT 电价标签 |
| 日前出清电价   | `日前出清电价` / `日前出清价格` / `DA出清电价` 等 | 可选特征 |
| 外生实际值     | `系统负荷实际值`、`光伏实际值`、`联络线实际值`、`上旋备用实际值`、`下旋备用实际值` 等 | 外生预测目标或 RT 特征 |
| 外生预测值     | 与“实际值”对应：如 `系统负荷预测值`、`光伏预测值` 等 | 外生方法 use_pred / ridge_residual 等会用到 |

- **格式**：Excel（.xlsx，首 sheet 或通过 `--sheet` 指定）或 CSV。
- **缺失值**：标签（如实时电价、外生实际值）不做插值，含 NaN 的样本会被丢弃；特征列可按 `--feat_nan_strategy` 做 ffill/bfill。

## 本目录在 Git 中的约定

- `data/README.md`、`data/schema.json` 随仓库提交。
- 大体积数据文件（如 `*.xlsx`、`*.csv`）已通过 `.gitignore` 排除，不随仓库推送；克隆后需自行将数据放入 `data/` 或通过 OSS/挂载提供。
